摘要

分享一下我在项目中用到的代码扫描工具。静态扫描工具Fortify和动态扫描工具Valgrind。

一、Valgrind

说明

Valgrind主要用于在程序运行时，监视内存的使用情况，是否存在内存泄漏等问题。

安装

Valgrind需要安装在程序运行环境上。安装过程如下：

  • 获取源码

wget http://www.valgrind.org/downloads/valgrind-3.18.1.tar.bz2

  • 编译安装
tar -jxvf valgrind-3.18.1.tar.bz2
cd valgrind-3.18.1
./configure --prefix=/home/valgrind
make
make install
  • 环境变量
vim ~/.bashrc
export PATH=$PATH:/home/valgrind/bin/
source ~/.bashrc

程序运行

valgrind –tool=memcheck –leak-check=yes –show-reachable=yes –run-libc-freeres=yes –log-file=./valgrind_report.log ./agt.out + 程序启动参数

扫描结果

Valgrind会报告5种级别的内存泄露：”definitely lost”,“indirectly lost”,“possibly lost”,“still reachable”,“suppressed”。通常情况下需要关注前三种级别，这里不再一一讲述每种级别的场景。

二、Fortify

说明

动态扫描工具的一个问题在于它依赖程序真实运行的环境，而一般真实运行环境不可能把所有的异常都跑到，所以需要引入静态扫描工具。Fortify是一款商用扫描工具，可以在代码编译期间对代码进行扫描，通过可视化的工具对各个代码异常点进行查看。

安装

  • 上传安装包至编译服务器/user/integration/autobuild/
  cd /user/integration/autobuild/
  mkdir fortify
  tar -zxf Fortify_SCA_and_Apps_4.40_Linux.tar.gz -C /user/integration/autobuild/fortify/

  • 拷贝License文件至/user/integration/autobuild/fortify/fortify.license

  • 执行安装，参考Fortify–安装与使用

  • 安装完成后拷贝规则文件至/opt/HP_Fortiy/HP_Fortify_SCA_and_Apps_4.40/Core/config/rules/

  • 顺便生成文件夹用于存放报告

  mkdir -p /user/integration/autobuild/fortify/Result/

编译

修改Makefile文件

vi output/Makefile.linux
CC == /opt/HP_Fortiy/HP_Fortify_SCA_and_Apps_4.40/bin/sourceanalyzer -b SAC $(TARGET)gcc
AR == /opt/HP_Fortiy/HP_Fortify_SCA_and_Apps_4.40/bin/sourceanalyzer -b SAC $(TARGET)ar

执行编译

执行编译后，Fortify会生成中间文件在/root/.fortify/sca6.4/build/SAC

最好每次扫描完成后备份该文件夹作为留存。

生成报告

/opt/HP_Fortiy/HP_Fortify_SCA_and_Apps_4.40/bin/sourceanalyzer -64 -Xmx4096M -Xms4096M -Xss24M -b SAC -scan -f /user/integration/autobuild/fortify/Result/SAC.results.fpr

报告生成后可以通过windows上安装的Fortify程序进行阅读
