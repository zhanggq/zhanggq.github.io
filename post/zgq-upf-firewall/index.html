UPF防火墙

UPF
约 441 字 预计阅读 1 分钟

    • 一、防火墙配置

摘要

在海外商用的环境被黑客攻破了，简单来说就是我们对外暴露的端口没有做防火墙，导致被人嗅探到了IP，且被破解了密码，最终被人ssh上去了。这里简单讲一下UPF与RHEL的firewalld的集成。

一、防火墙配置

协议

将需要放开的协议做成服务，放在/usr/lib/firewalld/services/目录下：

[root@testdept-upf-32-gsu-1 ~]# cat /usr/lib/firewalld/services/bgp.xml 
<?xml version="1.0" encoding="utf-8"?>
<service>
  <short>BGP</short>
  <description>BGP protocol.</description>
  <port protocol="tcp" port="179"/>
  <port protocol="tcp" port="36242"/>
  <port protocol="tcp" port="51832"/>
</service>
[root@testdept-upf-32-gsu-1 ~]# cat /usr/lib/firewalld/services/bfd.xml 
<?xml version="1.0" encoding="utf-8"?>
<service>
  <short>BFD</short>
  <description>BFD protocol.</description>
  <port protocol="udp" port="3784"/>
  <port protocol="udp" port="3785"/>
  <port protocol="udp" port="4784"/>
</service>

限制区

为每个对外接口增加一个限制区，该限制区默认为Drop，只放开部分需要的协议，放在/etc/firewalld/zones/目录下：

[root@testdept-upf-32-gsu-1 ~]# cat /etc/firewalld/zones/n3.xml 
<?xml version="1.0" encoding="utf-8"?>
<zone target="DROP">
  <short>Drop</short>
  <description>Unsolicited incoming network packets are dropped. Incoming packets that are related to outgoing network connections are accepted. Outgoing network connections are allowed.</description>
  <interface name="vEth0"/>
  <service name="bgp"/>
  <service name="bfd"/>
  <protocol value="ipv6-icmp"/>
  <protocol value="icmp"/>
  <protocol value="ospf"/>
</zone>
[root@testdept-upf-32-gsu-1 ~]# cat /etc/firewalld/zones/n6.xml 
<?xml version="1.0" encoding="utf-8"?>
<zone target="DROP">
  <short>Drop</short>
  <description>Unsolicited incoming network packets are dropped. Incoming packets that are related to outgoing network connections are accepted. Outgoing network connections are allowed.</description>
  <interface name="vEth1"/>
  <service name="bgp"/>
  <service name="bfd"/>
  <protocol value="ipv6-icmp"/>
  <protocol value="icmp"/>
  <protocol value="ospf"/>
</zone>

将系统默认区设置为trusted：

firewall-cmd --set-default-zone=trusted

其它

发现开启防火墙后ping6会不通，看有人也在说这个问题，解决方法是再加一条icmpv6的规则，参考Ipv6。

firewall-cmd --permanent --direct --add-rule ipv6 filter INPUT 0 -p icmpv6 -j ACCEPT
